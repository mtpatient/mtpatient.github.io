Java IO流原理及分类

原理

  1. I/O是Input/Outputl的缩写，I/O技术是非常实用的技术，用于处理数据传输。如读/写文件，网络通讯等。
  2. Java程序中，对于数据的输入/输出操作以”流(stream)”的方式进行。
  3. java.io包下提供了各种“流”类和接口，用以获取不同种类的数据，并通过方法输入或输出数据。
  4. 输入input：读取外部数据到内存当中。
  5. 输出output：将内存的数据输出到其它存储介质当中。

分类

  • 按操作数据单位不同分为：字节流(8bit 二进制文件) ,字符流（按字符 文本文件）

  • 按数据流的流向不同分为：输入流，输出流

  • 按流的角色的不同分为：节点流，处理流/包装流

    • 节点流：针对数据源读取数据，如FileInputStream、ByteArrayInputStream、pipedInputStream、StringReader等
    • 处理流/包装流：对节点流进行封装，消除不同节点流的实现差异，使用了修饰器设计模式

  • | 抽象基类 | 字节流 | 字符流 |
    | ———— | —————— | ——— |
    | 输入流 | InputStream | Reader |
    | 输出流 | OutPutStream | Writer |

InputStream：字节输入流

常用子类：

1.FileInputStream：文件输入流

单个字节读取：read()

每次只读取一个字节，返回-1表示读取完毕

读取到字节数组中：read(byte[] buf)

每次读取buf.length，返回实际读取字节数

2.BufferedInputStream：缓存字节输入流

字节流操作二进制文件

3. 对象流-ObjectInputStream

该类提供反序列化功能

涉及序列化和反序列化：

  • 序列化：将对象的值和数据类型分别保存到文件中，让某个类可序列化，该类及该类中的类成员变量必须得实现Serializable接口或Externalizable接口，序列化具备可继承性
    • Serializable：标记接口，仅作标记，没有方法需要实现
    • Externalizable：继承Serializable，需要实现方法
    • 序列化类时建议添加private static final long serialVersionUID = 1L;提高版本兼容性，static或transient修饰的成员不会被序列化
  • 反序列化：将保存在文件的数据重新恢复成对象

OutputStream：字节输出流

1.FileOutputStream：文件字节输出流

覆盖式写入：

write(byte[] b) 、write(byte[], int off, int len)

方法会覆盖文件的内容，write(byte[], int off, int len)将 len字节从位于偏移量off的指定字节数组写入此文件输出流。

若不想覆盖，则在创建FileOutputStream对象时，将append参数置为true

2.BufferedOutputStream：缓存字节输入流

3.对象流-ObjectOutputStream

提供序列化功能

Reader：字符输入流

1.FileReader

读取单个字符 read()

返回值为单个字符

读取多个字符 read(char[] chars)

返回值为实际读取字符数组的长度

2 .BufferedReader 处理流

Writer：字符输出流

1.FileWriter

写入操作后必须执行close()或flush()才能真正写入到磁盘当中， colse() = flush() + 关闭流

若不想覆盖，则在创建FileReader对象时，将append参数置为true

2. BufferedWriter

标准输入、标准输出（字节流）

System.in

  • 编译类型：InputStream
  • 运行类型：BufferedInputStream
  • 从键盘获取输入

System.out

  • 编译类型：PrintStream

  • 运行类型：PrintStream

转换流：InputStreamReader 和 OutputStreamWriter

可解决文件乱码问题，字节流转换成字符流

打印流：PrintStream 和 PrintReader

Properties类

读写配置文件， 底层为hashtable核心方法

  • load：加载配置文件的键值对到Properties对象
  • list：将数据显示到指定设备
  • getProperty()：根据键获取值
  • setProperty()：设置键值对到Properties对象
  • store：将Properties见制度存储到配置文件，如果含有中文，会转化为unicode编码
